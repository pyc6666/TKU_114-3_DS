## 問題情境

如果資料的先後順序由「下一筆是誰」決定，就能透過修改參考插入或移除節點。Linked List 將順序放在鏈結中，而不是固定索引中，因此必須清楚理解每次指定 next 後，哪些節點仍能從 head 找到。

## 核心概念

### 概念 1：為什麼需要鏈結串列

#### 概念說明

ArrayList 依索引保存資料，讀取指定位置很方便，但在前端或中間插入、刪除時，後方元素必須移動。Linked List 將資料分散在 Node 中，每個 Node 保存資料與下一個 Node 的參考。修改鏈結即可改變順序，不必搬移後方所有資料。

鏈結串列也有代價：它不能直接用索引跳到第 100 筆，通常必須從 head 一步一步走訪。實際選擇時要比較主要操作，而不是認為某一種結構永遠比較快。

#### 實際應用

- 工作流程中的下一項工作。
- 播放清單中的下一首歌曲。
- Queue 與 Deque 的底層鏈結概念。

#### 鏈結變化

陣列用連續位置表示順序；鏈結串列用 `next` 指向下一個 Node。

#### 何時適合使用

需要理解節點連接，或經常在已知節點後插入、刪除時。大量依索引查詢通常優先考慮 ArrayList。

#### 範例程式

檔名：`ListStructureComparison.java`

```java
import java.util.ArrayList;
import java.util.LinkedList;

public class ListStructureComparison {
    public static void main(String[] args) {
        ArrayList<String> arrayList = new ArrayList<>();
        LinkedList<String> linkedList = new LinkedList<>();

        arrayList.add("A");
        arrayList.add("B");
        linkedList.add("A");
        linkedList.add("B");

        arrayList.add(0, "Start");
        linkedList.addFirst("Start");

        System.out.println("ArrayList：" + arrayList);
        System.out.println("LinkedList：" + linkedList);
    }
}
```

執行結果：

```text
ArrayList：[Start, A, B]
LinkedList：[Start, A, B]
```

### 概念 2：Node 的 data、next 與 object reference

#### 概念說明

Node 是鏈結串列的基本單位。`data` 保存真正資料，`next` 保存下一個 Node 的參考。建立 Node 時，`next` 預設為 `null`，表示目前沒有連接下一個節點。

變數 `first` 與 `second` 保存的是物件參考，不是把整個 Node 複製進變數。執行 `first.next = second` 後，第一個 Node 能沿著 next 找到第二個 Node。

#### 實際應用

- 歌曲 Node 保存歌曲資料與下一首歌曲。
- 工作 Node 保存任務與下一項任務。
- 記憶體管理中的鏈結區塊。

#### 鏈結變化

連接前 `10 -> null`、`20 -> null`；連接後 `10 -> 20 -> null`。

#### 何時適合使用

需要建立可彼此連接的資料單位時。只保存單一固定資料不需要 Node。

#### 範例程式

檔名：`IntNode.java`

```java
public class IntNode {
    int data;
    IntNode next;

    public IntNode(int data) {
        this.data = data;
        this.next = null;
    }
}
```

執行結果：

```text

```

### 概念 3：head 與 null

#### 概念說明

`head` 保存第一個 Node 的參考，是進入整條串列的入口。當 `head == null` 時，串列沒有任何節點；只要失去 head 且沒有其他參考，就無法再從串列入口找到後方資料。

最後一個 Node 的 `next` 必須是 `null`，走訪時才能知道何時停止。`head` 不是資料本身，而是指向第一個節點的參考。

#### 實際應用

- 播放清單第一首。
- 待辦清單第一項。
- Queue 的最前端節點。

#### 鏈結變化

空串列：`head -> null`；加入 10 後：`head -> 10 -> null`。

#### 何時適合使用

所有單向鏈結串列都需要能找到起點。操作前先處理 head 為 null 的情況。

#### 範例程式

檔名：`HeadAndNullDemo.java`

```java
public class HeadAndNullDemo {
    public static void main(String[] args) {
        IntNode head = null;
        System.out.println("是否為空：" + (head == null));

        head = new IntNode(10);
        head.next = new IntNode(20);

        System.out.println("第一筆：" + head.data);
        System.out.println("第二筆：" + head.next.data);
        System.out.println("最後 next：" + head.next.next);
    }
}
```

執行結果：

```text

第一筆：10
第二筆：20
最後 next：null
```

### 概念 4：從 head 走訪整條串列

#### 概念說明

走訪時建立 `current` 指向目前節點。每次處理 `current.data` 後，執行 `current = current.next` 前往下一個 Node。當 current 變成 null，代表已經越過最後一個節點。

不要直接移動 head 來走訪，否則會改變串列入口。使用暫時的 current，可以在讀取完成後保留原本結構。

#### 實際應用

- 顯示完整播放清單。
- 計算所有節點總和。
- 搜尋指定工作。

#### 鏈結變化

current 依序指向 10、20、30，最後成為 null；head 始終指向 10。

#### 何時適合使用

需要逐筆讀取、統計或搜尋時。Linked List 沒有直接索引跳躍。

#### 範例程式

檔名：`LinkedListTraversalDemo.java`

```java
public class LinkedListTraversalDemo {
    public static void main(String[] args) {
        IntNode head = new IntNode(10);
        head.next = new IntNode(20);
        head.next.next = new IntNode(30);

        IntNode current = head;
        int total = 0;
        while (current != null) {
            System.out.println("目前資料：" + current.data);
            total += current.data;
            current = current.next;
        }

        System.out.println("總和：" + total);
        System.out.println("head 仍是：" + head.data);
    }
}
```

執行結果：

```text
目前資料：10
目前資料：20
目前資料：30
總和：60
head 仍是：10
```

#### 執行重點

執行前先畫出 head、current、previous 與 next 的指向。每完成一行改變參考的程式，就重新確認從 head 是否仍能走到所有應保留的節點。

### 概念 5：addFirst()：在前端新增

#### 概念說明

在前端新增需要兩個步驟：先讓新 Node 的 next 指向原本 head，再把 head 改為新 Node。順序不能顛倒；若先改 head，可能失去原本第一個 Node 的參考。

前端新增不需要走訪整條串列，不論目前有多少節點，操作步驟都固定。這是 Linked List 很典型的優勢。

#### 實際應用

- 插入緊急工作。
- 將最新通知放到最前面。
- Stack 的 push 操作。

#### 鏈結變化

原本 `head -> 20 -> 30`，加入 10 後變成 `head -> 10 -> 20 -> 30`。

#### 何時適合使用

新資料必須成為第一筆時。若要維持加入順序，應使用 addLast。

#### 範例程式

檔名：`AddFirstDemo.java`

```java
public class AddFirstDemo {
    public static void main(String[] args) {
        IntNode head = new IntNode(20);
        head.next = new IntNode(30);

        IntNode newNode = new IntNode(10);
        newNode.next = head;
        head = newNode;

        printList(head);
    }

    public static void printList(IntNode head) {
        IntNode current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }
}
```

執行結果：

```text
10 -> 20 -> 30 -> null
```

#### 執行重點

執行前先畫出 head、current、previous 與 next 的指向。每完成一行改變參考的程式，就重新確認從 head 是否仍能走到所有應保留的節點。

### 概念 6：addLast()：在尾端新增

#### 概念說明

尾端新增要先處理空串列；若 head 為 null，新 Node 直接成為 head。非空串列則從 head 走到 `current.next == null` 的最後一個 Node，再把它的 next 指向新 Node。

如果只保存 head，每次 addLast 都要走完整條串列。較完整的實作可以另外保存 tail，讓尾端新增不必重複走訪，但更新時也必須同時維護 head 與 tail。

#### 實際應用

- 依序加入播放清單。
- 把一般工作放到清單尾端。
- Queue 的 enqueue 操作。

#### 鏈結變化

`10 -> 20 -> null` 加入 30 後變成 `10 -> 20 -> 30 -> null`。

#### 何時適合使用

需要維持加入順序時。大量尾端新增可進一步加入 tail。

#### 範例程式

檔名：`AddLastDemo.java`

```java
public class AddLastDemo {
    public static void main(String[] args) {
        IntNode head = null;
        head = addLast(head, 10);
        head = addLast(head, 20);
        head = addLast(head, 30);
        printList(head);
    }

    public static IntNode addLast(IntNode head, int value) {
        IntNode newNode = new IntNode(value);
        if (head == null) {
            return newNode;
        }
        IntNode current = head;
        while (current.next != null) {
            current = current.next;
        }
        current.next = newNode;
        return head;
    }

    public static void printList(IntNode head) {
        IntNode current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }
}
```

執行結果：

```text
10 -> 20 -> 30 -> null
```

#### 執行重點

執行前先畫出 head、current、previous 與 next 的指向。每完成一行改變參考的程式，就重新確認從 head 是否仍能走到所有應保留的節點。

### 概念 7：搜尋、計數與 size

#### 概念說明

搜尋從 head 開始比較每個 current.data，找到後立即回傳 true；走到 null 仍未找到才回傳 false。若每次都走訪計算節點數量，size 查詢會重複花費時間，因此完整類別通常以 field 保存 size。

只要新增成功就 size++，刪除成功才 size--。size 與實際節點數不一致時，表示某個操作漏掉更新或錯誤更新。

#### 實際應用

- 確認歌曲是否存在。
- 顯示等待工作數。
- 檢查是否已加入某筆資料。

#### 鏈結變化

搜尋不改變鏈結；新增或刪除成功才改變 size。

#### 何時適合使用

鏈結串列中的循序搜尋。大量唯一鍵搜尋應考慮其他結構。

#### 範例程式

檔名：`SearchNodeDemo.java`

```java
public class SearchNodeDemo {
    public static void main(String[] args) {
        IntNode head = new IntNode(10);
        head.next = new IntNode(20);
        head.next.next = new IntNode(30);

        System.out.println("節點數：" + count(head));
        System.out.println("包含 20：" + contains(head, 20));
        System.out.println("包含 99：" + contains(head, 99));
    }

    public static int count(IntNode head) {
        int count = 0;
        IntNode current = head;
        while (current != null) {
            count++;
            current = current.next;
        }
        return count;
    }

    public static boolean contains(IntNode head, int target) {
        IntNode current = head;
        while (current != null) {
            if (current.data == target) {
                return true;
            }
            current = current.next;
        }
        return false;
    }
}
```

執行結果：

```text
節點數：3
包含 20：true
包含 99：false
```

執行前先畫出 head、current、previous 與 next 的指向。每完成一行改變參考的程式，就重新確認從 head 是否仍能走到所有應保留的節點。

### 概念 8：removeFirst()：刪除第一個節點

#### 概念說明

刪除第一個節點前先確認 head 是否為 null。非空時保存要回傳的資料，再執行 `head = head.next`，讓第二個節點成為新的第一個節點。

若串列只有一個節點，原本 head.next 就是 null，執行後 head 自然變成 null。相同程式可以同時處理單一節點與多節點。

#### 實際應用

- 完成第一項工作。
- 移除播放清單第一首。
- Queue 取出最前端資料。

#### 鏈結變化

`head -> 10 -> 20` 刪除第一筆後成為 `head -> 20`；只有 10 時則成為空串列。

#### 何時適合使用

需要移除最前端資料時。空串列必須回傳 null 或其他明確結果。

#### 範例程式

檔名：`RemoveFirstDemo.java`

```java
public class RemoveFirstDemo {
    public static void main(String[] args) {
        IntNode head = new IntNode(10);
        head.next = new IntNode(20);

        if (head != null) {
            int removed = head.data;
            head = head.next;
            System.out.println("刪除：" + removed);
        }

        System.out.println("新的 head：" + head.data);
    }
}
```

執行結果：

```text
刪除：10
新的 head：20
```

### 概念 9：依指定內容刪除節點

#### 概念說明

刪除指定節點需要同時知道 current 與 previous。找到 target 時，執行 `previous.next = current.next`，讓前一個節點跨過 current，直接連到下一個節點。

head 必須另外處理，因為第一個節點沒有 previous。找不到資料時不能改變串列，也不能錯誤減少 size。

#### 實際應用

- 取消指定工作。
- 從播放清單移除歌曲。
- 刪除指定會員紀錄。

#### 鏈結變化

`10 -> 20 -> 30` 刪除 20：把 10 的 next 從 20 改指向 30。

#### 何時適合使用

知道資料內容但沒有 Node 參考時，需要先走訪搜尋。

#### 範例程式

檔名：`RemoveValueDemo.java`

```java
public class RemoveValueDemo {
    public static void main(String[] args) {
        IntNode head = new IntNode(10);
        head.next = new IntNode(20);
        head.next.next = new IntNode(30);

        head = removeValue(head, 20);
        printList(head);
    }

    public static IntNode removeValue(IntNode head, int target) {
        if (head == null) {
            return null;
        }
        if (head.data == target) {
            return head.next;
        }
        IntNode previous = head;
        IntNode current = head.next;
        while (current != null) {
            if (current.data == target) {
                previous.next = current.next;
                break;
            }
            previous = current;
            current = current.next;
        }
        return head;
    }

    public static void printList(IntNode head) {
        IntNode current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }
}
```

執行結果：

```text
10 -> 30 -> null
```

#### 執行重點

執行前先畫出 head、current、previous 與 next 的指向。每完成一行改變參考的程式，就重新確認從 head 是否仍能走到所有應保留的節點。

### 概念 10：封裝成 IntLinkedList 類別

#### 概念說明

將 head 與 size 設為 private，可以避免外部程式直接破壞鏈結。外部只透過 addFirst、addLast、contains、removeFirst、removeValue 與 print 操作。

封裝後，呼叫端不需要知道 previous 與 current 如何移動。只要 method 的輸入、輸出與狀態規則穩定，內部實作可以繼續改進，例如加入 tail。

#### 實際應用

- 建立可重複使用的資料結構類別。
- 讓多個系統使用相同新增刪除邏輯。
- 將鏈結細節集中測試。

#### 鏈結變化

head 與 size 只由類別方法維護，外部無法直接指定錯誤鏈結。

#### 何時適合使用

資料結構需要重複使用時。不要把所有 Node 操作留在 main。

#### 範例程式

檔名：`IntLinkedList.java`

```java
public class IntLinkedList {
    private IntNode head;
    private int size;

    public IntLinkedList() {
        head = null;
        size = 0;
    }

    public int size() {
        return size;
    }

    public boolean isEmpty() {
        return head == null;
    }

    public void addFirst(int value) {
        IntNode newNode = new IntNode(value);
        newNode.next = head;
        head = newNode;
        size++;
    }

    public void addLast(int value) {
        IntNode newNode = new IntNode(value);
        if (head == null) {
            head = newNode;
            size++;
            return;
        }

        IntNode current = head;
        while (current.next != null) {
            current = current.next;
        }
        current.next = newNode;
        size++;
    }

    public boolean contains(int target) {
        IntNode current = head;
        while (current != null) {
            if (current.data == target) {
                return true;
            }
            current = current.next;
        }
        return false;
    }

    public Integer removeFirst() {
        if (head == null) {
            return null;
        }
        int removed = head.data;
        head = head.next;
        size--;
        return removed;
    }

    public boolean removeValue(int target) {
        if (head == null) {
            return false;
        }
        if (head.data == target) {
            head = head.next;
            size--;
            return true;
        }

        IntNode previous = head;
        IntNode current = head.next;
        while (current != null) {
            if (current.data == target) {
                previous.next = current.next;
                size--;
                return true;
            }
            previous = current;
            current = current.next;
        }
        return false;
    }

    public void reverse() {
        IntNode previous = null;
        IntNode current = head;

        while (current != null) {
            IntNode nextNode = current.next;
            current.next = previous;
            previous = current;
            current = nextNode;
        }
        head = previous;
    }

    public void print() {
        IntNode current = head;
        while (current != null) {
            System.out.print(current.data + " -> ");
            current = current.next;
        }
        System.out.println("null");
    }
}
```

### 概念 11：空串列、單一節點與邊界情況

#### 概念說明

Linked List 最常出錯的地方不是一般多節點，而是空串列與單一節點。每個操作都要問：head 為 null 時怎麼辦？刪除唯一節點後 head 是否正確變成 null？找不到資料時 size 是否保持不變？

先寫出邊界案例再實作，比完成後才補救更可靠。資料結構程式應至少測試空、單一、多筆、找不到與重複值。

#### 實際應用

- 剛建立但尚無資料的系統。
- 只剩最後一筆工作。
- 刪除不存在資料時保持狀態。

#### 鏈結變化

單一節點刪除後：`head -> 10 -> null` 變成 `head -> null`，size 由 1 變 0。

#### 何時適合使用

所有資料結構操作都應先定義邊界行為。

#### 範例程式

### 概念 12：反轉單向鏈結串列

#### 概念說明

反轉時不能直接把 current.next 改掉後再尋找下一個節點，因為原本的 next 參考會遺失。每一輪要先保存 nextNode，再讓 current.next 指向 previous，最後將 previous 與 current 向前移動。

迴圈結束時，previous 指向原本最後一個節點，它應成為新的 head。這個操作同時練習 object reference 與狀態追蹤。

#### 實際應用

- 反向顯示操作紀錄。
- 調整處理順序。
- 常見程式面試與參考操作。

#### 鏈結變化

`10 -> 20 -> 30 -> null` 反轉後為 `30 -> 20 -> 10 -> null`。

#### 何時適合使用

確實需要改變節點方向時。只想反向輸出時，也可以考慮 Stack，避免修改原結構。

#### 範例程式

檔名：`LinkedListReverseDemo.java`

```java
public class LinkedListReverseDemo {
    public static void main(String[] args) {
        IntLinkedList list = new IntLinkedList();
        list.addLast(10);
        list.addLast(20);
        list.addLast(30);

        System.out.print("反轉前：");
        list.print();
        list.reverse();
        System.out.print("反轉後：");
        list.print();
    }
}
```

執行結果：

```text
反轉前：10 -> 20 -> 30 -> null
反轉後：30 -> 20 -> 10 -> null
```

### 概念 13：ArrayList 與 Linked List 的操作選擇

#### 概念說明

ArrayList 適合依索引讀取與尾端新增；Linked List 適合在已知節點附近改變鏈結。若 Linked List 仍要先從 head 搜尋目標，整體操作依然需要走訪，不能只看到刪除就認為一定比較快。

在一般 Java 應用中，ArrayList 常是預設選擇；Queue 或 Stack 通常優先使用 ArrayDeque。自訂 Linked List 的主要價值是理解 Node、參考與資料結構內部運作。

#### 實際應用

- 動態表格資料通常使用 ArrayList。
- Queue 與 Deque 需要頻繁操作兩端。
- LRU cache 結合雙向鏈結與 Map。

#### 何時適合使用

## 常見錯誤與診斷

- 直接移動 head 走訪，導致失去串列入口。
- 改變 current.next 前沒有先保存 nextNode。
- addLast 沒有處理空串列。
- removeValue 沒有另外處理 head。
- 找不到資料仍然減少 size。
- 刪除唯一節點後 head 沒有變成 null。
- 只背程式碼，無法畫出 previous、current 與 next 的變化。

## 評分規準

- Node 與鏈結方向正確。
- 新增、搜尋、刪除與反轉符合規格。
- head、size 與空串列狀態一致。
- method 拆分與封裝清楚。
- 所有檔名與 public class 一致，程式可共同編譯。
