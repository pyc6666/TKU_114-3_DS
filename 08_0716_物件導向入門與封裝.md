# 7/16 教材：物件導向入門與封裝

## 單元名稱

Java `class`、`object`、建構子、實例方法、封裝與物件陣列

## 課程定位

0714 與 0715 使用商品名稱、價格、庫存三個平行陣列保存資料。這種方式可以完成程式，但必須確保相同索引永遠代表同一項商品。當欄位增加時，陣列數量與方法參數會持續增加。

物件導向程式設計 `Object-Oriented Programming, OOP` 會把相關資料與操作放進同一個類別。例如一個 `Product` 物件同時擁有名稱、價格、庫存，以及購買、補貨與計算庫存價值的方法。

## 學習目標

完成本單元後，應能：

1. 說明 `class` 與 `object` 的關係。
2. 在類別中宣告欄位。
3. 使用建構子建立具有初始資料的物件。
4. 使用 `this` 區分欄位與參數。
5. 建立並呼叫實例方法。
6. 分辨 `static` 方法與實例方法。
7. 使用 `private`、getter 與 setter 完成封裝。
8. 在方法中維護物件資料的合法狀態。
9. 覆寫 `toString()`。
10. 建立、走訪與搜尋物件陣列。
11. 將商品平行陣列重構成 `Product[]`。
12. 將 `0716` 資料夾 push 到 GitHub 並繳交正確連結。

## 問題情境

平行陣列版本的商品資料：

```java
String[] names = {"Keyboard", "Mouse"};
int[] prices = {890, 490};
int[] stocks = {12, 20};
```

商品資料分散在三個陣列中。若排序時只交換名稱，價格與庫存就會對錯商品。方法也常需要同時接收三個陣列：

```java
printProducts(names, prices, stocks);
```

物件版本可以改成：

```java
Product[] products = {
    new Product("Keyboard", 890, 12),
    new Product("Mouse", 490, 20)
};
```

每個陣列元素都是一個完整商品，資料與操作不再分散。

### 概念 2：`class` 與 `object`

#### 範例程式

檔名：`ProductBasics.java`、`ProductBasicsDemo.java`

類別 `class` 是物件的設計規格，物件 `object` 是依類別建立出的實體。

```java
public class ProductBasics {
    String name;
    int price;
    int stock;
}
```

`ProductBasics` 描述商品有哪些資料，但尚未建立任何商品物件。

建立物件：

```java
ProductBasics keyboard = new ProductBasics();
```

`new` 會建立物件，`keyboard` 保存該物件的參考。

### 概念 3：欄位 field

#### 範例程式

檔名：`ProductBasics.java`、`ProductBasicsDemo.java`（延續概念 2）

類別中的變數稱為欄位或實例變數：

```java
public class ProductBasics {
    String name;
    int price;
    int stock;
}
```

每一個物件都有自己的一份欄位：

```java
ProductBasics keyboard = new ProductBasics();
keyboard.name = "Keyboard";
keyboard.price = 890;
keyboard.stock = 12;

ProductBasics mouse = new ProductBasics();
mouse.name = "Mouse";
mouse.price = 490;
mouse.stock = 20;
```

修改 `keyboard.stock` 不會影響 `mouse.stock`。

### 概念 4：建構子 constructor

#### 範例程式

檔名：`Book.java`、`BookDemo.java`

建構子會在 `new` 物件時執行，用來建立合法的初始狀態。

```java
public class Book {
    String title;
    int price;

    public Book(String title, int price) {
        this.title = title;
        this.price = price;
    }
}
```

建構子特徵：

- 名稱與類別名稱相同。
- 沒有回傳型態，連 `void` 都不寫。
- 可以有參數。
- 呼叫 `new Book(...)` 時自動執行。

建立物件：

```java
Book javaBook = new Book("Java Basics", 650);
```

### 概念 5：`this`

#### 範例程式

檔名：`Book.java`、`BookDemo.java`（延續概念 4）

建構子參數與欄位同名時，使用 `this` 表示目前物件：

```java
public Book(String title, int price) {
    this.title = title;
    this.price = price;
}
```

- `title`：建構子的參數。
- `this.title`：目前物件的欄位。

沒有 `this` 時，`title = title` 只會把參數指定給自己，欄位不會得到資料。

### 概念 6：實例方法

#### 範例程式

檔名：`Rectangle.java`、`RectangleDemo.java`

物件除了保存資料，也可以提供操作：

```java
public class Rectangle {
    double width;
    double height;

    public double calculateArea() {
        return width * height;
    }
}
```

呼叫實例方法前必須先建立物件：

```java
Rectangle rectangle = new Rectangle();
rectangle.width = 5;
rectangle.height = 3;

System.out.println(rectangle.calculateArea());
```

方法會使用呼叫它的物件資料。

### 概念 7：`static` 與實例成員

#### 範例程式

檔名：`Rectangle.java`、`RectangleDemo.java`（延續概念 6）

前面寫在主程式類別中的方法大多使用 `static`：

```java
public static int calculateTotal(int[] values)
```

`static` 成員屬於類別，可直接用類別名稱或在 `main` 中呼叫。實例方法屬於物件：

```java
product.sell(2);
```

判斷方式：

- 操作特定物件狀態：使用實例方法。
- 不依賴任何特定物件的工具功能：可使用 `static` 方法。
- `main` 必須是 `static`，因為程式啟動時還沒有主程式類別的物件。

### 概念 8：封裝 encapsulation

#### 範例程式

檔名：`BankAccount.java`、`BankAccountDemo.java`

若欄位是 `public` 或未限制，外部程式可以直接寫入錯誤資料：

```java
account.balance = -50000;
```

封裝會把欄位設為 `private`，只允許透過方法操作：

```java
public class BankAccount {
    private String owner;
    private int balance;
}
```

`private` 欄位只能在同一個類別內直接存取。

### 概念 9：getter

#### 範例程式

檔名：`BankAccount.java`、`BankAccountDemo.java`（延續概念 8）

getter 用來讀取私有欄位：

```java
public String getOwner() {
    return owner;
}

public int getBalance() {
    return balance;
}
```

getter 通常不接收參數，名稱以 `get` 開頭並回傳欄位值。

### 概念 10：setter 與資料驗證

#### 範例程式

檔名：`Product.java`

setter 用來修改私有欄位，並可在修改前驗證：

```java
public void setPrice(int price) {
    if (price > 0) {
        this.price = price;
    }
}
```

不是每個欄位都一定需要 setter。銀行餘額不應提供任意修改的 `setBalance()`，應透過 `deposit()` 與 `withdraw()` 維護規則。

### 概念 11：用方法維護合法狀態

#### 範例程式

檔名：`BankAccount.java`、`BankAccountDemo.java`（延續概念 8）

```java
public boolean withdraw(int amount) {
    if (amount <= 0 || amount > balance) {
        return false;
    }

    balance -= amount;
    return true;
}
```

回傳 `boolean` 讓呼叫端知道操作是否成功。物件自己負責保證餘額不會變成負數。

### 概念 12：覆寫 `toString()`

#### 範例程式

檔名：`Book.java`、`BookDemo.java`（延續概念 4）

所有 Java 物件都有 `toString()`。可以覆寫成適合顯示的格式：

```java
@Override
public String toString() {
    return name + "，價格：" + price + "，庫存：" + stock;
}
```

之後直接輸出物件：

```java
System.out.println(product);
```

`@Override` 讓編譯器檢查方法是否真的覆寫父類別方法。

### 概念 13：物件參考

#### 範例程式

檔名：`ObjectReferenceDemo.java`

物件變數保存的是參考：

```java
Product first = new Product("Mouse", 490, 20);
Product second = first;

second.restock(5);
System.out.println(first.getStock()); // 25
```

`first` 與 `second` 指向同一個物件。這與陣列變數的參考行為相同。

### 概念 14：物件陣列

#### 範例程式

檔名：`ProductArrayDemo.java`

建立最多可放 5 個商品參考的陣列：

```java
Product[] products = new Product[5];
```

此時每個元素都是 `null`，必須逐一建立物件：

```java
products[0] = new Product("Keyboard", 890, 12);
products[1] = new Product("Mouse", 490, 20);
```

也可以直接初始化：

```java
Product[] products = {
    new Product("Keyboard", 890, 12),
    new Product("Mouse", 490, 20)
};
```

### 概念 15：走訪與搜尋物件陣列

#### 範例程式

檔名：`ProductArrayDemo.java`（延續概念 14）

```java
for (Product product : products) {
    if (product != null) {
        System.out.println(product);
    }
}
```

依名稱搜尋：

```java
public static Product findProduct(
        Product[] products, int count, String name) {
    for (int i = 0; i < count; i++) {
        if (products[i].getName().equalsIgnoreCase(name.trim())) {
            return products[i];
        }
    }
    return null;
}
```

找不到時回傳 `null`。呼叫方法前必須先判斷結果是否為 `null`。

### 概念 16：從平行陣列重構為物件陣列

#### 範例程式

檔名：`Product.java`、`ProductManagementSystem.java`

原本：

```java
String[] names;
int[] prices;
int[] stocks;
```

重構後：

```java
Product[] products;
```

購買功能原本需要同時修改庫存陣列，重構後改由商品物件處理：

```java
if (product.sell(quantity)) {
    System.out.println("購買成功");
} else {
    System.out.println("數量錯誤或庫存不足");
}
```

## 常見錯誤與診斷

| 常見錯誤 | 原因 | 修正方式 |
|---|---|---|
| `constructor ... cannot be applied` | 建構子參數數量或型態不符 | 對照建構子宣告 |
| 建構子欄位仍是預設值 | 寫成 `name = name` | 使用 `this.name = name` |
| `non-static ... cannot be referenced` | 在 `static main` 直接呼叫實例成員 | 先建立物件再呼叫 |
| 外部無法存取欄位 | 欄位為 `private` | 使用 getter 或類別提供的方法 |
| `NullPointerException` | 對 `null` 元素呼叫方法 | 先檢查物件是否存在 |
| 物件陣列輸出只有類別名稱與代碼 | 未覆寫 `toString()` | 在類別中覆寫 `toString()` |
| 失敗轉帳仍扣除餘額 | 操作順序錯誤 | 驗證全部條件後再修改兩個物件 |
| 主程式仍有大量商品規則 | 類別責任切分不清 | 將庫存與價格規則移到 `Product` |
