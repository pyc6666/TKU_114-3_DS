# 7/16 教材：物件導向入門與封裝

## 單元名稱

Java `class`、`object`、建構子、實例方法、封裝與物件陣列

## 課程定位

0714 與 0715 使用商品名稱、價格、庫存三個平行陣列保存資料。這種方式可以完成程式，但必須確保相同索引永遠代表同一項商品。當欄位增加時，陣列數量與方法參數會持續增加。

物件導向程式設計 `Object-Oriented Programming, OOP` 會把相關資料與操作放進同一個類別。例如一個 `Product` 物件同時擁有名稱、價格、庫存，以及購買、補貨與計算庫存價值的方法。

## 問題情境

平行陣列版本的商品資料：

```java
String[] names = {"Keyboard", "Mouse"};
int[] prices = {890, 490};
int[] stocks = {12, 20};
```

商品資料分散在三個陣列中。若排序時只交換名稱，價格與庫存就會對錯商品。方法也常需要同時接收三個陣列：

```java
printProducts(names, prices, stocks);
```

物件版本可以改成：

```java
Product[] products = {
    new Product("Keyboard", 890, 12),
    new Product("Mouse", 490, 20)
};
```

每個陣列元素都是一個完整商品，資料與操作不再分散。

### 概念 2：`class` 與 `object`

類別 `class` 是物件的設計規格，物件 `object` 是依類別建立出的實體。

```text
public class ProductBasics {
    String name;
    int price;
    int stock;
}
```

`ProductBasics` 描述商品有哪些資料，但尚未建立任何商品物件。

建立物件：

```text
ProductBasics keyboard = new ProductBasics();
```

`new` 會建立物件，`keyboard` 保存該物件的參考。

#### 範例程式

檔名：`ProductBasics.java`

```java
public class ProductBasics {
    String name;
    int price;
    int stock;
}
```

檔名：`ProductBasicsDemo.java`

```java
public class ProductBasicsDemo {
    public static void main(String[] args) {
        ProductBasics keyboard = new ProductBasics();
        keyboard.name = "Keyboard";
        keyboard.price = 890;
        keyboard.stock = 12;

        ProductBasics mouse = new ProductBasics();
        mouse.name = "Mouse";
        mouse.price = 490;
        mouse.stock = 20;

        System.out.println(keyboard.name + "，價格：" + keyboard.price);
        System.out.println(mouse.name + "，庫存：" + mouse.stock);
    }
}
```

### 概念 4：建構子 constructor

建構子會在 `new` 物件時執行，用來建立合法的初始狀態。

```text
public class Book {
    String title;
    int price;

    public Book(String title, int price) {
        this.title = title;
        this.price = price;
    }
}
```

建構子特徵：

- 名稱與類別名稱相同。
- 沒有回傳型態，連 `void` 都不寫。
- 可以有參數。
- 呼叫 `new Book(...)` 時自動執行。

建立物件：

```text
Book javaBook = new Book("Java Basics", 650);
```

#### 範例程式

檔名：`Book.java`

```java
public class Book {
    private String title;
    private int price;

    public Book(String title, int price) {
        this.title = title;
        this.price = price;
    }

    public String getTitle() {
        return title;
    }

    public int getPrice() {
        return price;
    }

    @Override
    public String toString() {
        return title + "，價格：" + price;
    }
}
```

檔名：`BookDemo.java`

```java
public class BookDemo {
    public static void main(String[] args) {
        Book first = new Book("Java Basics", 650);
        Book second = new Book("Data Structures", 720);

        System.out.println(first);
        System.out.println(second);
        System.out.println("第一本書名：" + first.getTitle());
    }
}
```

### 概念 5：`this`

建構子參數與欄位同名時，使用 `this` 表示目前物件：

```text
public Book(String title, int price) {
    this.title = title;
    this.price = price;
}
```

- `title`：建構子的參數。
- `this.title`：目前物件的欄位。

沒有 `this` 時，`title = title` 只會把參數指定給自己，欄位不會得到資料。

#### 範例程式

```java
public class Book {

        return title;

    public int getPrice() {
        return price;
    }

    @Override
    public String toString() {
        return title + "，價格：" + price;
    }
}
```

檔名：`BookDemo.java`


### 概念 6：實例方法

物件除了保存資料，也可以提供操作：

```text
public class Rectangle {
    double width;
    double height;

    public double calculateArea() {
        return width * height;
    }
}
```

呼叫實例方法前必須先建立物件：

```text
Rectangle rectangle = new Rectangle();
rectangle.width = 5;
rectangle.height = 3;

System.out.println(rectangle.calculateArea());
```

方法會使用呼叫它的物件資料。

#### 範例程式

檔名：`Rectangle.java`

```java
public class Rectangle {
    private double width;
    private double height;

    public Rectangle(double width, double height) {
        setWidth(width);
        setHeight(height);
    }

    public double getWidth() {
        return width;
    }

    public double getHeight() {
        return height;
    }

    public boolean setWidth(double width) {
        if (width <= 0) {
            return false;
        }
        this.width = width;
        return true;
    }

    public boolean setHeight(double height) {
        if (height <= 0) {
            return false;
        }
        this.height = height;
        return true;
    }

    public double calculateArea() {
        return width * height;
    }

    public double calculatePerimeter() {
        return 2 * (width + height);
    }

    public boolean isSquare() {
        return Double.compare(width, height) == 0;
    }

    @Override
    public String toString() {
        return "寬：" + width + "，高：" + height;
    }
}
```

檔名：`RectangleDemo.java`

```java
public class RectangleDemo {
    public static void printResult(Rectangle rectangle) {
        System.out.println(rectangle);
        System.out.printf("面積：%.2f%n", rectangle.calculateArea());
        System.out.printf("周長：%.2f%n", rectangle.calculatePerimeter());
        System.out.println("正方形：" + rectangle.isSquare());
        System.out.println();
    }

    public static void main(String[] args) {
        Rectangle first = new Rectangle(5, 3);
        Rectangle second = new Rectangle(4, 4);
        Rectangle third = new Rectangle(8.5, 2);

        printResult(first);
        printResult(second);
        printResult(third);
    }
}
```

### 概念 7：`static` 與實例成員

前面寫在主程式類別中的方法大多使用 `static`：

```text
public static int calculateTotal(int[] values)
```

`static` 成員屬於類別，可直接用類別名稱或在 `main` 中呼叫。實例方法屬於物件：

```text
product.sell(2);
```

判斷方式：

- 操作特定物件狀態：使用實例方法。
- 不依賴任何特定物件的工具功能：可使用 `static` 方法。
- `main` 必須是 `static`，因為程式啟動時還沒有主程式類別的物件。

#### 範例程式

檔名：`Rectangle.java`

```java

    private double height;

        setWidth(width);
        setHeight(height);

        return width;
    }

    public double getHeight() {
        return height;
    }

    public boolean setWidth(double width) {
        if (width <= 0) {
            return false;
        }
        this.width = width;
        return true;
    }

    public boolean setHeight(double height) {
        if (height <= 0) {
            return false;
        }
        this.height = height;
        return true;
    }

    public double calculateArea() {
        return width * height;
    }

    public double calculatePerimeter() {
        return 2 * (width + height);
    }

    public boolean isSquare() {
        return Double.compare(width, height) == 0;
    }

    @Override
    public String toString() {
```

### 概念 8：封裝 encapsulation

若欄位是 `public` 或未限制，外部程式可以直接寫入錯誤資料：

```text
account.balance = -50000;
```

封裝會把欄位設為 `private`，只允許透過方法操作：

```text
public class BankAccount {
    private String owner;
    private int balance;
}
```

`private` 欄位只能在同一個類別內直接存取。

#### 範例程式

檔名：`BankAccount.java`

```java
public class BankAccount {
    private String accountNumber;
    private String owner;
    private int balance;

    public BankAccount(
            String accountNumber, String owner, int initialBalance) {
        this.accountNumber = accountNumber;
        this.owner = owner;
        this.balance = Math.max(initialBalance, 0);
    }

    public String getAccountNumber() {
        return accountNumber;
    }

    public String getOwner() {
        return owner;
    }

    public int getBalance() {
        return balance;
    }

    public boolean deposit(int amount) {
        if (amount <= 0) {
            return false;
        }
        balance += amount;
        return true;
    }

    public boolean withdraw(int amount) {
        if (amount <= 0 || amount > balance) {
            return false;
        }
        balance -= amount;
        return true;
    }

    public boolean transferTo(BankAccount target, int amount) {
        if (target == null || target == this
                || amount <= 0 || amount > balance) {
            return false;
        }

        balance -= amount;
        target.balance += amount;
        return true;
    }

    @Override
    public String toString() {
        return accountNumber + "，戶名：" + owner + "，餘額：" + balance;
    }
}
```

檔名：`BankAccountDemo.java`

```java
public class BankAccountDemo {
    public static void main(String[] args) {
        BankAccount first = new BankAccount("A001", "Amy", 5000);
        BankAccount second = new BankAccount("A002", "Ben", 1000);

        System.out.println("存款成功：" + first.deposit(500));
        System.out.println("提款成功：" + second.withdraw(300));
        System.out.println("轉帳成功：" + first.transferTo(second, 2000));
        System.out.println("超額轉帳：" + second.transferTo(first, 10000));

        System.out.println(first);
        System.out.println(second);
    }
}
```

### 概念 10：setter 與資料驗證

setter 用來修改私有欄位，並可在修改前驗證：

```text
public void setPrice(int price) {
    if (price > 0) {
        this.price = price;
    }
}
```

不是每個欄位都一定需要 setter。銀行餘額不應提供任意修改的 `setBalance()`，應透過 `deposit()` 與 `withdraw()` 維護規則。

#### 範例程式

檔名：`Product.java`

```java
public class Product {
    private String name;
    private int price;
    private int stock;

    public Product(String name, int price, int stock) {
        if (name == null || name.trim().isEmpty()) {
            this.name = "Unnamed";
        } else {
            this.name = name.trim();
        }

        this.price = Math.max(price, 1);
        this.stock = Math.max(stock, 0);
    }

    public String getName() {
        return name;
    }

    public int getPrice() {
        return price;
    }

    public int getStock() {
        return stock;
    }

    public boolean setPrice(int price) {
        if (price <= 0) {
            return false;
        }
        this.price = price;
        return true;
    }

    public boolean restock(int quantity) {
        if (quantity <= 0) {
            return false;
        }
        stock += quantity;
        return true;
    }

    public boolean sell(int quantity) {
        if (quantity <= 0 || quantity > stock) {
            return false;
        }
        stock -= quantity;
        return true;
    }

    public boolean isLowStock() {
        return stock < 10;
    }

    public long getInventoryValue() {
        return (long) price * stock;
    }

    @Override
    public String toString() {
        return name + "，價格：" + price + "，庫存：" + stock;
    }
}
```

檔名：`ProductDemo.java`

```java
public class ProductDemo {
    public static void main(String[] args) {
        Product product = new Product("Keyboard", 890, 12);

        System.out.println("原始資料：" + product);

        if (product.setPrice(920)) {
            System.out.println("價格修改成功");
        }

        if (product.restock(5)) {
            System.out.println("補貨成功");
        }

        if (product.sell(3)) {
            System.out.println("出售成功");
        }

        System.out.println("更新後：" + product);
        System.out.println("低庫存：" + product.isLowStock());
        System.out.println("庫存價值：" + product.getInventoryValue());
    }
}
```

### 概念 11：用方法維護合法狀態

```text
public boolean withdraw(int amount) {
    if (amount <= 0 || amount > balance) {
        return false;

    balance -= amount;
    return true;
```


### 概念 12：覆寫 `toString()`

所有 Java 物件都有 `toString()`。可以覆寫成適合顯示的格式：

```text
@Override
public String toString() {
    return name + "，價格：" + price + "，庫存：" + stock;
}
```

之後直接輸出物件：


### 概念 13：物件參考

物件變數保存的是參考：

```text
Product first = new Product("Mouse", 490, 20);
Product second = first;

second.restock(5);
System.out.println(first.getStock()); // 25
```

`first` 與 `second` 指向同一個物件。這與陣列變數的參考行為相同。

#### 範例程式

檔名：`Product.java`

```java
public class Product {
    private String name;
    private int price;
    private int stock;

    public Product(String name, int price, int stock) {
        if (name == null || name.trim().isEmpty()) {
            this.name = "Unnamed";
        } else {
            this.name = name.trim();
        }

        this.price = Math.max(price, 1);
        this.stock = Math.max(stock, 0);
    }

    public String getName() {
        return name;
    }

    public int getPrice() {
        return price;
    }

    public int getStock() {
        return stock;
    }

    public boolean setPrice(int price) {
        if (price <= 0) {
            return false;
        }
        this.price = price;
        return true;
    }

    public boolean restock(int quantity) {
        if (quantity <= 0) {
            return false;
        }
        stock += quantity;
        return true;
    }

    public boolean sell(int quantity) {
        if (quantity <= 0 || quantity > stock) {
            return false;
        }
        stock -= quantity;
        return true;
    }

    public boolean isLowStock() {
        return stock < 10;
    }

    public long getInventoryValue() {
        return (long) price * stock;
    }

    @Override
    public String toString() {
        return name + "，價格：" + price + "，庫存：" + stock;
    }
}
```

```java
                return product;
            }
```

## 常見錯誤與診斷

| 常見錯誤 | 原因 | 修正方式 |
|---|---|---|
| `constructor ... cannot be applied` | 建構子參數數量或型態不符 | 對照建構子宣告 |
| 建構子欄位仍是預設值 | 寫成 `name = name` | 使用 `this.name = name` |
| `non-static ... cannot be referenced` | 在 `static main` 直接呼叫實例成員 | 先建立物件再呼叫 |
| 外部無法存取欄位 | 欄位為 `private` | 使用 getter 或類別提供的方法 |
| `NullPointerException` | 對 `null` 元素呼叫方法 | 先檢查物件是否存在 |
| 物件陣列輸出只有類別名稱與代碼 | 未覆寫 `toString()` | 在類別中覆寫 `toString()` |
| 失敗轉帳仍扣除餘額 | 操作順序錯誤 | 驗證全部條件後再修改兩個物件 |
| 主程式仍有大量商品規則 | 類別責任切分不清 | 將庫存與價格規則移到 `Product` |
