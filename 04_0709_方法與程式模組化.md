# 7/9 教材：方法與程式模組化

## 單元名稱

Java 方法 `method`、參數、回傳值、變數範圍與程式模組化

## 課程定位

7/8 已經能使用 `while`、`switch`、累加器、計數器與輸入驗證做出點餐系統或 ATM 系統初版。當程式變長後，如果所有程式都寫在 `main` 裡，會很難閱讀、修改與除錯。

本單元開始學習方法 `method`。方法可以把一段有明確目的的程式碼取出來，給它一個名稱，之後需要時再呼叫。

今天的重點是：

1. 看懂方法的基本結構。
2. 使用沒有參數、沒有回傳值的方法。
3. 使用有參數的方法。
4. 使用有回傳值的方法。
5. 理解變數範圍 scope。
6. 把 7/8 的選單式程式重構成多個方法。

## 問題情境

如果把整個點餐系統都寫在 `main` 裡，程式會快速變長：

- 顯示選單是一段程式。
- 讀取選項是一段程式。
- 檢查數量是一段程式。
- 計算小計是一段程式。
- 印出收據又是一段程式。

當所有事情都擠在 `main` 裡，常見問題是：

- 找不到錯誤在哪一段。
- 想改選單時，不小心改到計算邏輯。
- 同樣的程式碼重複出現。
- `main` 太長，不容易閱讀。

本單元要解決的問題是：

> 如何把一個比較長的程式拆成多個清楚的小功能？

### 概念 2：什麼是 method

#### 概念說明

方法 `method` 是一段有名稱的程式碼。把程式碼放進方法後，可以在需要時呼叫它。

方法的好處：

1. 讓 `main` 變短。
2. 讓每段程式有明確目的。
3. 減少重複程式碼。
4. 讓除錯更容易。
5. 讓程式更容易修改與擴充。

基本格式：

```java
public static void methodName() {
    方法內要執行的程式
}
```

先記住：

- `methodName` 是方法名稱。
- `void` 代表這個方法不回傳資料。
- 方法要被呼叫才會執行。

#### 範例程式

檔名：`MethodDemo.java`

```java
public class MethodDemo {
    public static void main(String[] args) {
        sayHello();
    }

    public static void sayHello() {
        System.out.println("Hello Java");
    }
}
```

### 概念 3：沒有參數、沒有回傳值的方法

#### 概念說明

如果某段程式只是固定執行一件事，不需要外部資料，也不需要回傳結果，可以寫成沒有參數、沒有回傳值的方法。

例如顯示選單：

```java
public static void printMenu() {
    System.out.println("1. Start");
    System.out.println("0. Exit");
}
```

呼叫方式：

```java
printMenu();
```

#### 範例程式

檔名：`MenuMethodDemo.java`

```java
public class MenuMethodDemo {
    public static void main(String[] args) {
        printMenu();
    }

    public static void printMenu() {
        System.out.println("=== Menu ===");
        System.out.println("1. Start");
        System.out.println("0. Exit");
    }
}
```

#### 課堂實作題

建立 `PrintOrderMenu.java`，寫一個 `printOrderMenu()` 方法，輸出：

```text
=== Order Menu ===
1. Black tea
2. Green tea
3. Coffee
0. Checkout
```

### 概念 4：有參數的方法

#### 概念說明

參數 parameter 是呼叫方法時傳進去的資料。方法可以根據傳入的資料做不同事情。

基本格式：

```java
public static void methodName(資料型態 參數名稱) {
    使用參數
}
```

例如：

```java
public static void greet(String name) {
    System.out.println("Hello, " + name);
}
```

呼叫時傳入資料：

```java
greet("Amy");
greet("Ben");
```

同一個方法可以處理不同資料，這就是參數的價值。

#### 範例程式

檔名：`ParameterDemo.java`

```java
public class ParameterDemo {
    public static void main(String[] args) {
        greet("Amy");
        greet("Ben");
    }

    public static void greet(String name) {
        System.out.println("Hello, " + name);
    }
}
```

#### 課堂實作題

建立 `PricePrinter.java`，寫一個方法：

```java
public static void printItem(String itemName, int price)
```

呼叫後輸出商品名稱與價格。

### 概念 5：有回傳值的方法

#### 概念說明

如果方法負責計算，通常需要把結果回傳給呼叫它的地方。這時候就不能使用 `void`，要把 `void` 改成回傳資料型態。

基本格式：

```java
public static 回傳型態 methodName(參數) {
    return 回傳值;
}
```

例如：

```java
public static int add(int a, int b) {
    return a + b;
}
```

呼叫時可以把結果存起來：

```java
int result = add(3, 5);
```

#### 範例程式

檔名：`ReturnDemo.java`

```java
public class ReturnDemo {
    public static void main(String[] args) {
        int result = add(3, 5);
        System.out.println("Result: " + result);
    }

    public static int add(int a, int b) {
        return a + b;
    }
}
```

#### 課堂實作題

建立 `SubtotalCalculator.java`，寫一個方法：

```java
public static int calculateSubtotal(int price, int quantity)
```

回傳商品小計。

### 概念 6：BMI 方法拆解

#### 概念說明

7/2 的 BMI 程式可以拆成兩個方法：

1. `calculateBmi()`：負責計算 BMI。
2. `getBmiLevel()`：負責根據 BMI 判斷分級。

這樣 `main` 就不用同時處理計算公式與分級規則。

#### 範例程式

檔名：`BmiMethod.java`

```java
public class BmiMethod {
    public static void main(String[] args) {
        double height = 1.65;
        double weight = 55;

        double bmi = calculateBmi(height, weight);
        String level = getBmiLevel(bmi);

        System.out.println("BMI: " + bmi);
        System.out.println("Level: " + level);
    }

    public static double calculateBmi(double height, double weight) {
        return weight / (height * height);
    }

    public static String getBmiLevel(double bmi) {
        if (bmi < 18.5) {
            return "Underweight";
        } else if (bmi < 24) {
            return "Normal";
        } else if (bmi < 27) {
            return "Overweight";
        } else {
            return "Obese";
        }
    }
}
```

### 概念 7：變數範圍 scope

#### 概念說明

變數範圍 scope 是指變數在哪裡可以被使用。

在方法裡宣告的變數，只能在該方法裡使用，這種變數稱為 local variable。

例如：

```java
public static void methodA() {
    int number = 10;
}

public static void methodB() {
    System.out.println(number); // 錯誤：number 不在這個方法的範圍內
}
```

如果另一個方法需要資料，應該透過參數傳入，或透過回傳值取回結果。

#### 範例程式

檔名：`ScopeDemo.java`

```java
public class ScopeDemo {
    public static void main(String[] args) {
        int score = 80;
        printScore(score);
    }

    public static void printScore(int score) {
        System.out.println("Score: " + score);
    }
}
```

#### 課堂實作題

### 概念 8：方法命名與拆分原則

#### 概念說明

方法名稱應該清楚描述它做的事情。常見命名方式：

| 動作          | 方法名稱範例          |
| ------------- | --------------------- |
| 顯示選單      | `printMenu()`         |
| 計算小計      | `calculateSubtotal()` |
| 計算平均      | `calculateAverage()`  |
| 判斷等第      | `getGrade()`          |
| 判斷 BMI 分級 | `getBmiLevel()`       |
| 讀取合法數量  | `readValidQuantity()` |

拆方法時可以用這些問題判斷：

1. 這段程式是否有明確目的？
2. 這段程式是否重複出現？
3. 這段程式是否讓 `main` 太長？
4. 這段程式是否可以用名稱說清楚？

#### 範例：點餐系統可拆成的方法

```java
printMenu()
getPrice(int option)
getItemName(int option)
readValidQuantity(Scanner sc)
calculateSubtotal(int price, int quantity)
printReceipt(int totalItems, int totalAmount)
```

### 概念 9：重構 refactor

#### 概念說明

重構 refactor 是指在不改變功能的前提下，整理程式結構，讓程式更容易閱讀、維護與擴充。

重構前：

- 所有程式都寫在 `main`。
- 同樣的選單文字重複出現。
- 計算邏輯和輸出混在一起。

重構後：

- `main` 只保留主要流程。
- 每個方法負責一件事。
- 方法名稱能說明功能。

重構不是重寫整個程式，而是一步一步把程式拆乾淨。

#### 範例流程

1. 找出顯示選單的程式碼。
2. 搬到 `printMenu()`。
3. 在原本位置呼叫 `printMenu()`。
4. 執行程式確認功能沒有壞掉。
5. 再拆下一段。

## 常見錯誤與診斷

| 常見錯誤                    | 可能原因                                       | 診斷方式                                          | 修正方式                              |
| --------------------------- | ---------------------------------------------- | ------------------------------------------------- | ------------------------------------- |
| 方法寫了但沒有執行          | 沒有在`main` 呼叫方法                          | 程式執行後沒有任何效果                            | 在需要的位置呼叫方法                  |
| 回傳型態不一致              | 方法宣告是`int`，但回傳 `String` 或沒有 return | 編譯錯誤顯示 missing return 或 incompatible types | 確認回傳型態與`return` 的值一致       |
| `void` 方法中回傳資料       | `void` 代表不回傳值                            | 編譯錯誤                                          | 若需要回傳，將`void` 改成正確型態     |
| 變數在方法中找不到          | 變數超出 scope                                 | 出現 cannot find symbol                           | 用參數傳入需要的資料                  |
| 方法名稱不清楚              | 方法做的事情太多或名稱太模糊                   | 看名稱不知道用途                                  | 使用動詞開頭，例如`calculateSubtotal` |
| `Scanner` 傳遞混亂          | 每個方法都重新 new Scanner                     | 輸入行為不穩定或程式難管理                        | 建議把`Scanner sc` 當參數傳入方法     |
| 重構後功能壞掉              | 一次改太多                                     | 原本功能突然不能用                                | 每拆一個方法就執行測試                |
| iClass 只貼 repository 首頁 | 沒有貼到當天資料夾                             | 連結打開後看不到`0709` 內容                       | 貼上`.../tree/main/0709` 形式的連結   |
